Pasos Generales

A continuación se muestra un flujo de trabajo básico para usar Git y GitHub:

  1. Inicializar el repositorio local con git init.
  2. Crear o modificar archivos en tu proyecto.
  3. Agregar los cambios al área de preparación con git add.
  4. Guardar los cambios en el historial con git commit.
  5. Vincular tu repositorio local con uno remoto usando git remote add origin <URL>.
  6. Enviar tus cambios al repositorio remoto con git push.
  7. Recibir cambios del repositorio remoto con git pull.

Comandos Básicos de Git

Comando Descripción
git init Inicializa un nuevo repositorio local.
git add [archivo] Añade un archivo o cambios al área de preparación (staging).
git commit -m "Mensaje" Guarda los cambios en el historial con un mensaje descriptivo.
git push Sube los commits locales al repositorio remoto.
git pull Descarga los cambios más recientes del repositorio remoto.

Ejemplo de Código

Ejemplo rápido de una secuencia de comandos en Git:

git init
git add .
git commit -m "Primer commit"
git remote add origin https://github.com/user/repo.git
git push -u origin main

Para más información, puedes visitar la Documentación Oficial de Git.
